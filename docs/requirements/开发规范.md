# DevTools Station - 开发规范

## 代码风格

### Java
- 使用 Lombok 简化 POJO
- Controller -> Service -> Mapper 分层架构
- 统一使用 `Result<T>` 作为 API 响应格式
- 遵循阿里巴巴 Java 开发手册

### 前端
- 使用 Thymeleaf 模板引擎
- CSS 变量实现主题切换
- 原生 JS，无框架依赖
- 响应式设计（Mobile First 补充）

## 添加新工具的步骤

### 1. 如果是纯前端工具
直接在 `src/main/resources/templates/tools/` 下创建 HTML 模板即可。

### 2. 如果需要后端处理
1. 在对应的 `service` 包中创建服务类
2. 在对应的 `controller` 包中创建控制器
3. 在 `docs/sql/init.sql` 中添加工具数据（如需要数据库记录）

### 3. 注册路由
在 `PageController.java` 中添加路由映射，或在 Thymeleaf 中直接链接。

## Git 提交规范
```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```
